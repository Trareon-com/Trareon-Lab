# Synthetic NTFS corpus (ntfs-synth-v1)

| Field | Value |
|---|---|
| Class | synthetic |
| Format | `TRFSNT01` binary (`*.trfsnt`) |
| Purpose | Day 16 NTFS **enumeration** semantics without redistributing a full NTFS volume image |
| Redistribution | allowed (generated; recipe below) |
| Sensitivity | none |

## Recipe

1. Build entries with `lab_fs::write_ntfs_synthetic_corpus`.
2. Root directory uses MFT-like record number `5`.
3. Paths are reconstructed with `\` separators from parent links.
4. Flags: `0x1` directory, `0x2` file, `0x4` deleted.

Demo corpus path: `validation/synthetic/ntfs/ntfs-synth-demo.trfsnt` (regenerated by `cargo test -p lab-fs --test ntfs_enumerate`).

## Non-claims

- Not a full NTFS boot sector / $MFT attribute parser.
- Alternate Data Streams (ADS) not encoded in v1.
- Real raw NTFS volume support requires a governed public/restricted image under `docs/CORPUS-GOVERNANCE.md`.
